# Estrutura do Banco de Atividades do ACESSA+

## Objetivo

Construir um acervo pedagogico profissional, organizado, pesquisavel e reutilizavel, com materiais prontos para aplicacao em AEE, sala comum, apoio pedagogico, formacao docente e uso familiar.

O banco nao deve guardar apenas atividades soltas. Ele deve organizar conjuntos editoriais por habilidade curricular e perfil AEE.

## Organizacao principal

```text
Etapa de ensino
> Ano/serie
> Disciplina
> Habilidade
> Perfil AEE
> Atividade 01
> Atividade 02
> Atividade 03
> Atividade 04
> Atividade 05
```

## Exemplo


## Fase 1 - disciplinas prioritarias

Ensino Fundamental II, 6 ao 9 Ano:

- Lingua Portuguesa;
- Matematica;
- Ciencias;
- Historia;
- Geografia.

Essas areas devem ser priorizadas por concentrarem a maior demanda de adaptacao curricular no AEE.

## Fase 2 - disciplinas complementares

- Ingles;
- Arte;
- Educacao Fisica;
- Ensino Religioso.

## Perfis obrigatorios por habilidade

Cada habilidade deve gerar conjuntos especificos para:

- DI;
- TEA;
- DV;
- DA;
- TDAH;
- AH/SD;
- Multiplas necessidades.

## Padrao editorial de cada material

Cada material deve conter:

- titulo curto;
- etapa;
- ano/serie;
- disciplina;
- codigo da habilidade;
- objeto de conhecimento;
- perfil AEE;
- nivel de apoio;
- nivel de leitura;
- folha A4 pronta;
- CAA quando pertinente;
- recursos tateis quando pertinente;
- Libras/Braille/audiodescricao quando pertinente;
- tags de busca;
- status editorial.

## Status editorial

```text
RASCUNHO
EM_REVISAO
PUBLICADO
ARQUIVADO
```

## Origem do material

```text
OFICIAL_ACESSA
GERADO_IA
ENVIADO_USUARIO
REVISADO_EDITORIAL
```

## Cadastro de usuarios

O cadastro deve guardar:

- nome completo;
- email;
- contato/WhatsApp;
- instituicao ou rede;
- municipio;
- estado;
- area de atuacao;
- aceite para contato futuro.

Esses dados permitem comunicacao posterior para grupo de WhatsApp, Telegram, suporte, formacao e comunidade de professores.

## Regra de privacidade

O banco de atividades nao deve armazenar dados pessoais de estudantes.

Materiais devem usar somente campos genericos:

```text
NOME: ________________________
TURMA: _______________________
DATA: ____ / ____ / ______
DISCIPLINA: _________________
PROFESSOR(A): _______________
```
